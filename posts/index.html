Peer to Peer Git

Mar 31, 2017

Sometimes your central git repository goes down, but you need to share code. That’s a bummer, you can commit and wait, but you’ve likely heard that you shouldn’t need to rely on a central repo. So you’ve got a few options here: Mirror your repo someplace else and switch upstreams during outages, email your stuff as patches using [git am][2], [2]: https://git-scm.com/docs/git-am setup a small read only local git server and pull between neighbours, or use some kind of highly available decentralised storage as an upstream like [ipfs git][4].
